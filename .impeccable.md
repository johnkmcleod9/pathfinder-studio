# Impeccable Design Context — Pathfinder Studio

> Source of truth for every visual decision in this codebase.
> Established via the Impeccable `teach` flow, 2026-04-17.
> Impeccable's AI Slop Test is the bar: every design choice here
> exists to ensure a reviewer would ask "how was this made?" not
> "which AI made this?"

## Design Context

### Users

Two distinct audiences, both needing to be respected as
intelligent professionals:

1. **Course authors** — the Pathwise Solutions internal team
   (instructional designers, LMS developers) building `.pathfinder`
   sources. They work in VS Code / terminal / file editors today;
   any authoring surface we build later (Phase 5+) needs to feel
   like a pro tool, not a drag-and-drop toy.

2. **Course learners** — adult professionals in three primary
   contexts:
   - **Canadian government employees** completing mandatory or
     role-based training, often during work hours at a desk,
     often via an LMS they didn't choose.
   - **Healthcare workers** (nurses, clinicians, admin staff)
     completing CE, compliance, or clinical-update modules —
     frequently late shift, often on tablets, often interrupted.
   - **Corporate staff** completing onboarding, compliance,
     upskilling — at a desk during work hours.

The job-to-be-done for learners: *get through this module with
my intelligence and time respected, retain something useful,
move on.* They are not looking for gamified fun; they are looking
for a training artifact that treats them as a peer.

### Brand Personality

**Crisp · Editorial · Grown-up.**

- **Crisp** — every decision visible and intentional. Sharp
  execution. Nothing muddy, nothing hedged. Edges have reasons.
- **Editorial** — magazine-level composition. Confident
  typographic hierarchy, intentional negative space, asymmetry
  where it earns its keep, refined color. Feels curated, not
  templated.
- **Grown-up** — treats the reader as an intelligent
  professional. No confetti on completion. No "Great job! 🎉"
  microcopy. No cartoon mascots. No patronizing feedback tone.

Emotional goals: **respected, focused, finished on time.**

### Aesthetic Direction

**Lineage: Craft · Bear · Ghost, pulled toward editorial
precision.** Think Phaidon or Princeton Architectural Press
meets a modern productivity app. Think *The Browser Company*
but printed on good paper stock.

Concrete anchors:
- **Warm paper surfaces**, not plastic chrome. Neutrals tinted
  toward the brand hue so the whole interface feels materially
  cohesive.
- **Serif for content, sans for UI.** The course body text (the
  thing the learner actually reads) lives in a distinctive
  reading-optimized serif. Interface chrome (buttons, labels,
  nav, quiz option metadata) lives in a refined sans. Both
  chosen to reject the training-data monoculture (see Font
  Constraints below).
- **Confident hierarchy.** Fewer sizes with greater contrast
  (≥1.25 ratio). Heading weight carries the hierarchy; body
  type stays restrained.
- **Color used sparingly and with meaning.** 60% neutral
  surface · 30% ink · 10% accent. One dominant brand hue with
  a sharp supporting accent; rare, never decorative.

### Theme Strategy

**Ship both light and dark from day one.** Each designed
intentionally (not auto-inverted).

- **Light theme** — warm paper white, ink-tinted-toward-brand-
  hue text. Feels like a well-produced educational print
  artifact. Default for day-at-a-desk learners.
- **Dark theme** — deep ink-tinted charcoal background,
  warm-cream text. Feels like evening reading on quality
  stock. Serves night-shift healthcare, long-module scenarios,
  and learners who've opted in via system preference.

Respect `prefers-color-scheme`; also provide an explicit toggle
for explicit user override. Both themes share the same token
names — only the values change.

### Anti-References (what this must NEVER resemble)

All four explicitly rejected:

1. **Articulate Storyline / Rise.** No default blue CTAs. No
   bottom-right nav arrows pretending to be navigation. No
   drop-shadow cards. No "template-y" interactivity vocabulary
   (exclamation-mark popups, green-check completion chimes,
   yellow-highlight hints).

2. **Generic corporate LMS** (Cornerstone, Absorb, SAP
   SuccessFactors). No pastel clipart cards. No confetti on
   completion. No "Certificate!" modal. No fake gamification
   (leaderboards, XP bars, unlockable badges as the primary
   motivator). No 90s-workbook feel.

3. **AI-dashboard aesthetic.** No cyan-on-dark. No purple-to-
   blue gradients. No neon-glowing accents. No glassmorphism
   as decoration. No sparklines-as-decoration. No rounded
   rectangles with generic drop shadows. No gradient text (per
   Impeccable's absolute ban).

4. **Cartoony e-learning for adults.** No Comic Sans energy.
   No Bitmoji-style mascots. No exaggerated rounded corners.
   No elementary-classroom visual metaphors.

## Design Principles (the 5 rules that govern every pixel)

1. **Editorial clarity over template uniformity.** Every slide
   is composed with intent — hierarchy, asymmetry where useful,
   deliberate negative space. No "card grid repeated forever."
   Not everything needs a container.

2. **Warm materiality over flat neutrality.** Surfaces feel like
   paper or fabric. Neutrals are tinted toward the brand hue
   (≥0.005 chroma in OKLCH). No pure `#fff` or `#000` — both
   themes use tinted near-whites / near-blacks.

3. **Serif for reading, sans for interface.** A distinctive
   reading-optimized serif carries the learner's actual content.
   A refined sans carries UI chrome. Both chosen from outside
   the AI-reflex font list.

4. **Respect the professional learner.** Microcopy is precise,
   information-dense, honest. No confetti. No "Great job!"
   Feedback tells the learner what they got right or wrong
   and why, in the same register a good colleague would use.

5. **Motion is meaning, not decoration.** Exponential easing
   (ease-out-quart/expo). Transform + opacity only. Never
   animate layout properties. Honor `prefers-reduced-motion`
   rigorously. Motion exists for state changes, entrances,
   exits, focus feedback — nothing else.

## Hard Constraints

### Typography
- **Reject** all fonts on Impeccable's `reflex_fonts_to_reject`
  list: Fraunces, Newsreader, Lora, Crimson (+Pro/Text),
  Playfair Display, Cormorant (+Garamond), Syne, IBM Plex
  {Mono, Sans, Serif}, Space {Mono, Grotesk}, Inter, DM {Sans,
  Serif Display, Serif Text}, Outfit, Plus Jakarta Sans,
  Instrument {Sans, Serif}.
- Pair a distinctive serif (body + possibly display) with a
  refined UI sans.
- ≥1.25 modular type scale, 5 steps is plenty.
- Body prose capped at 65–75ch measure.
- Line-height scales inversely with column width. For light
  text on dark background, add 0.05–0.1 to normal leading.
- Fluid clamp() sizing on headings for marketing/content
  contexts (e.g., standalone tour); fixed rem scale for
  in-course UI chrome.

### Color
- **Brand hue: tobacco / burnt sienna** —
  `oklch(0.48 0.11 45)` at its fullest expression. A warm
  earth-red with brown undertone. Reference neighbours:
  Moleskine classic covers, vintage library cloth bindings,
  Japanese washi dyed with persimmon. Every neutral in both
  themes is tinted toward this hue (≈0.005–0.01 chroma).
- **OKLCH only** in the CSS (hex values permitted as inline
  documentation of the OKLCH tokens, but `color:` properties
  use `oklch()` functions).
- Reduce chroma at extreme lightness — a near-white with 0.15
  chroma looks garish.
- WCAG AA contrast minimum for all text; AAA for body prose
  where feasible. (Govt requirement; non-negotiable.)
- Never pure `#fff` or `#000`.
- Never gradient text (per Impeccable absolute ban).
- Never >1px side-stripe borders on cards/callouts (per
  Impeccable absolute ban).

### Spacing & Layout
- 4pt base grid. Semantic token names
  (`--space-sm`, `--space-md`, etc.), not pixel-named.
- `gap` for sibling spacing, not margins.
- Asymmetry is encouraged where it earns emphasis.
- Slide canvas is 1280×720 (compiler default) for Phase 1b;
  scroll mode (Phase 3) will go fluid with container queries.
- Not every block is a card. Flatten hierarchy where possible.

### Motion
- Exponential ease-out curves only (quart / quint / expo).
- Transform + opacity only — never animate layout props.
- Honor `prefers-reduced-motion` rigorously; collapse to
  instant state changes, not shortened motion.

### Accessibility (WCAG 2.1 AA floor, AAA where feasible)
- Keyboard operable: every interactive surface focusable in
  logical order, `:focus-visible` ring always present and
  high-contrast.
- ARIA roles and labels already part of the runtime (see
  `src/publish/browser-runtime.ts` — `role="region"`,
  `aria-live="polite"` on slides; `fieldset` + `legend` on
  quiz questions). Preserve and extend.
- Minimum 44×44px interactive targets.
- Screen-reader announcements on slide change + quiz result.
- Color-only is never the sole signal (icon + text + color).
- Respect system font size; support browser zoom to 200%.
